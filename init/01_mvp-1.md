본 문서는 Convention CLI 프로젝트의 1차 MVP 개발을 위한 오케스트레이션 계획서이다.

5. Agent별 작업 카드 형태
아래처럼 각각의 AI Agent에게 작업을 던질 수 있습니다.


작업 카드 1
Agent: 프로젝트 초기화 Agent


목표:
Node.js 기반 convention-cli 프로젝트의 기본 폴더와 파일을 생성한다.


해야 할 일:
1. package.json 생성
2. bin/convention.js 생성
3. src/commands, src/core, src/config, src/utils 폴더 생성
4. README.md 생성
5. .gitignore 생성


완료 조건:
node bin/convention.js 실행 시 에러 없이 기본 메시지가 출력되어야 한다.




작업 카드 2
Agent: npm 전역 설치 Agent


목표:
npm link 또는 npm install -g 이후 convention 명령어가 실행되도록 한다.


해야 할 일:
1. package.json에 bin 필드 추가
2. bin/convention.js 상단에 #!/usr/bin/env node 추가
3. convention --help 실행 가능하도록 연결


완료 조건:
npm link 후 convention --help 명령어가 실행되어야 한다.




작업 카드 3
Agent: 설정 저장 Agent


목표:
사용자 설정을 ~/.config/convention/config.json에 저장하고 불러온다.


해야 할 일:
1. 기본 설정값 정의
2. 설정 폴더 자동 생성
3. 설정 파일이 없으면 기본값 반환
4. 설정 파일이 있으면 JSON 파싱
5. 설정 저장 함수 구현


완료 조건:
saveConfig 후 loadConfig를 실행하면 저장한 값이 그대로 반환되어야 한다.




작업 카드 4
Agent: --set-mode Agent


목표:
convention --set-mode step 또는 batch 명령어를 구현한다.


해야 할 일:
1. 입력값 검증
2. 기존 config 로드
3. mode 값 변경
4. config 저장
5. 성공 메시지 출력


완료 조건:
convention --set-mode batch 실행 후 config.json의 mode 값이 batch로 변경되어야 한다.




작업 카드 5
Agent: --language Agent


목표:
convention --language ko/en/jp/cn 명령어를 구현한다.


해야 할 일:
1. 입력값 검증
2. 기존 config 로드
3. language 값 변경
4. config 저장
5. 성공 메시지 출력


완료 조건:
convention --language en 실행 후 config.json의 language 값이 en으로 변경되어야 한다.




작업 카드 6
Agent: Git diff Agent


목표:
현재 Git 저장소의 변경 사항과 diff를 추출한다.


해야 할 일:
1. Git 저장소 여부 확인
2. 변경 파일 목록 가져오기
3. 전체 diff 가져오기
4. 파일별 diff 가져오기
5. UTF-8과 core.quotepath=false 적용


완료 조건:
변경된 파일 목록과 diff 문자열을 정상 반환해야 한다.




작업 카드 7
Agent: Prompt 생성 Agent


목표:
Git diff를 기반으로 AI 커밋 메시지 생성용 프롬프트를 만든다.


해야 할 일:
1. diff 입력받기
2. language 입력받기
3. mode 입력받기
4. Conventional Commits 규칙 포함
5. 커밋 메시지만 반환하라는 조건 포함


완료 조건:
buildCommitPrompt 실행 시 AI에게 보낼 완성된 프롬프트 문자열이 반환되어야 한다.




작업 카드 8
Agent: AI 메시지 생성 Agent


목표:
프롬프트를 AI Provider에 보내고 커밋 메시지를 받아온다.


해야 할 일:
1. config에서 provider 정보 읽기
2. fetch로 요청 보내기
3. 응답에서 메시지 추출
4. 응답 정리
5. 빈 응답 예외 처리


완료 조건:
generateCommitMessage(prompt) 실행 시 커밋 메시지 문자열이 반환되어야 한다.




작업 카드 9
Agent: Batch 커밋 Agent


목표:
전체 변경 사항을 하나의 커밋으로 만든다.


해야 할 일:
1. Git 저장소 확인
2. 변경 사항 확인
3. 전체 diff 추출
4. prompt 생성
5. AI 메시지 생성
6. git add -A
7. git commit -m 실행


완료 조건:
convention --batch 실행 시 전체 변경 사항이 하나의 커밋으로 생성되어야 한다.




작업 카드 10
Agent: Step 커밋 Agent


목표:
변경 파일별로 각각 커밋을 생성한다.


해야 할 일:
1. Git 저장소 확인
2. 변경 파일 목록 확인
3. 파일별 diff 추출
4. 파일별 prompt 생성
5. 파일별 AI 메시지 생성
6. 파일별 git add
7. 파일별 git commit


완료 조건:
convention --step 실행 시 파일별 커밋이 생성되어야 한다.




작업 카드 11
Agent: 기본 convention 실행 Agent


목표:
옵션 없이 convention을 실행했을 때 config.mode에 따라 step 또는 batch를 실행한다.


해야 할 일:
1. config 로드
2. mode 확인
3. step이면 runStepCommit 호출
4. batch이면 runBatchCommit 호출
5. mode가 없으면 step 기본값 사용


완료 조건:
convention 실행 시 설정된 기본 모드대로 커밋이 진행되어야 한다.
