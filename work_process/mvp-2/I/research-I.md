# Phase I API Key 입력 Agent Research

## 1. 개요

Phase I는 사용자가 API Key를 안전하게 입력할 수 있도록 지원하는 단계입니다. 이 단계는 화면에 입력값이 노출되지 않는 프롬프트를 구현하고, 입력된 값을 검증한 후 credentials 저장 로직과 연결하는 것을 목표로 합니다.

## 2. 작업 목표

- 비밀번호 입력 형태(masking)로 API Key 입력 구현
- 빈 값 등 유효하지 않은 입력 검증
- 입력 과정 중 화면 및 일반 로그 출력 금지
- 검증된 API Key를 credentials 저장 함수로 전달하는 흐름 구성

## 3. 권장 구현 방향

- `prompts` 라이브러리의 `type: 'password'` 또는 `type: 'invisible'` 옵션을 사용하여 터미널 화면에 API Key가 직접 노출되지 않도록 합니다.
- 사용자가 값을 입력하지 않고 엔터를 치거나 취소(Ctrl+C)하는 경우에 대한 예외 처리 및 에러 안내 로직을 작성합니다.
- 입력받은 API Key는 반환값으로만 전달하며, 절대로 `console.log` 등을 통해 로깅하지 않습니다.

## 4. 보안 및 안정성 기준

- 어떠한 상황에서도(오류 발생 포함) 입력중인 또는 입력된 API Key 원문이 터미널 표준 출력(stdout)이나 에러 출력(stderr)에 남지 않아야 합니다.
- 입력된 키는 메모리에 최소한의 시간 동안만 보관되며, 즉시 credentials 저장 로직으로 전달되어야 합니다.

## 5. 다음 단계 연결

안전하게 입력 및 검증된 API Key는 Phase J에서 다룰 Credentials 저장 Agent를 통해 시스템에 암호화/분리 저장되어야 합니다.
