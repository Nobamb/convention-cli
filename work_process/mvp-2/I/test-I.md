# Phase I API Key 입력 Agent Test

`research-I.md` 기준으로 정의된 API Key 입력 로직이 화면과 로그에 노출 없이 안전하게 처리되는지 검증하기 위한 테스트 항목입니다.

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :--- | :--- | :--- | :--- |
| I-V-1 | 프롬프트 마스킹 처리 | 코드 내 `prompts` 옵션 확인 | `type: 'password'` 또는 그와 동등한 마스킹 처리가 적용되어 있다. |
| I-V-2 | 로깅 금지 확인 | API Key 변수 사용 부분 확인 | 입력값을 직접 `console.log` 또는 `logger.info` 등으로 출력하는 코드가 없다. |

## 2. 기능 테스트 항목

### I-T-1: 정상 입력 및 마스킹

- **준비:** CLI에서 API Key 입력 프롬프트 강제 실행
- **실행:** API Key를 타이핑한다.
- **예상 결과:** 타이핑하는 동안 화면에 실제 문자가 아닌 `*` 처리 또는 아예 문자가 보이지 않으며, 입력 완료 후 다음 로직으로 정상 전달된다.

### I-T-2: 빈 값 입력 처리

- **준비:** CLI에서 API Key 입력 프롬프트 실행
- **실행:** 아무것도 입력하지 않고 엔터를 입력한다.
- **예상 결과:** API Key가 비어있다는 경고 메시지가 출력되고 재입력을 요구하거나 적절한 에러를 발생시키며 종료된다.

## 3. 검증 결과 요약

- **모든 항목 통과 시:** API Key 입력 단계가 보안 요구사항을 충족하며 성공적으로 구현됨.
- **실패 항목 존재 시:** 입력 마스킹 누락 또는 로깅 여부를 점검한다.
