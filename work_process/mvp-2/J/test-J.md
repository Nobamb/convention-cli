# Phase J Credentials 저장 Agent Test

`research-J.md` 기준으로 정의된 자격 증명 저장 로직이 일반 설정 파일과 분리되어 안전한 권한으로 저장되는지 검증하기 위한 테스트 항목입니다.

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :--- | :--- | :--- | :--- |
| J-V-1 | 분리 저장 아키텍처 | 코드 내 저장 경로 확인 | API Key는 `config.json`이 아닌 `credentials.json`에만 저장되도록 작성되어 있다. |
| J-V-2 | 파일 권한 설정 | 파일 쓰기 옵션 확인 | `fs.writeFile` 등에 `mode: 0o600` 등 제한된 권한이 설정되어 있다. |

## 2. 기능 테스트 항목

### J-T-1: Credentials 정상 저장

- **준비:** 특정 Provider(예: gemini)에 대해 가상의 API Key 준비
- **실행:** API Key를 `credentials.json`에 저장하는 함수 호출
- **예상 결과:** `~/.config/convention/credentials.json` 파일이 성공적으로 생성 또는 업데이트되며, 해당 내용에 저장한 API Key가 포함되어 있다.

### J-T-2: 파일 권한 확인 (Linux/macOS)

- **준비:** J-T-1을 통해 `credentials.json`이 생성된 상태
- **실행:** 파일 시스템에서 생성된 파일의 권한을 확인 (`ls -l` 등)
- **예상 결과:** 소유자만 읽기/쓰기가 가능한 권한(600, `-rw-------`)으로 설정되어 있다.

### J-T-3: Config 파일 분리 확인

- **준비:** J-T-1 실행 완료 후
- **실행:** `config.json` 파일 내용 점검
- **예상 결과:** `config.json` 내에는 어떠한 API Key 정보도 존재하지 않는다.

## 3. 검증 결과 요약

- **모든 항목 통과 시:** 자격 증명이 보안 기준을 준수하며 성공적으로 분리 저장됨.
- **실패 항목 존재 시:** 저장 경로 꼬임 또는 권한 부여 로직을 점검한다.
