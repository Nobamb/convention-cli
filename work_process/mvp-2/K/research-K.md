# Phase K API Key 보안 Agent Research

## 1. 개요

Phase K는 프로그램이 실행되는 전 과정에서 API Key 등 민감한 인증 정보가 실수로 노출되지 않도록 전역적인 보안 장치 및 마스킹 로직을 구현하는 단계입니다. 로그 출력, 에러 메시지, 그리고 JSON 덤프 과정 등에서 키가 유출되는 것을 원천 차단합니다.

## 2. 작업 목표

- 로그 출력 전 API Key 문자열을 탐지하여 마스킹(`***`) 처리하는 유틸리티 구현
- 에러 객체 출력 시 메시지에 API Key가 포함되는지 검사 및 필터링
- `JSON.stringify` 호출 시 credentials 객체가 통째로 출력되지 않도록 제한
- 개발자/사용자 디버그 모드에서도 민감 정보 출력 차단

## 3. 권장 구현 방향

- 커스텀 Logger 모듈에 필터 파이프라인을 추가하여, 설정된 API Key들의 일부 또는 전체가 콘솔에 찍히기 전에 마스킹되도록 구성합니다.
- 외부 API 호출 실패로 인한 에러 메시지(Axios/Fetch 에러 등)에 쿼리 파라미터나 헤더의 Authorization 내용이 그대로 출력되는 경우가 많으므로, 이를 잡아서 안전한 메시지로 치환합니다.

## 4. 보안 및 안정성 기준

- 어떠한 예기치 않은 오류(네트워크 타임아웃, 잘못된 응답 파싱 등)에서도 시스템 표준 출력(stdout/stderr)에 인증 토큰이 원문 그대로 나타나서는 안 됩니다.

## 5. 다음 단계 연결

Phase K를 통해 안전한 실행 환경이 보장되면, Phase L과 M에서 실제 외부 API로 요청을 전송하는 클라우드 Provider 연동을 안심하고 진행할 수 있습니다.
