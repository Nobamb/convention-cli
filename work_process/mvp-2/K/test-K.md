# Phase K API Key 보안 Agent Test

`research-K.md` 기준으로 정의된 보안 및 마스킹 로직이 로그, 에러 등 모든 출력 경로에서 API Key 노출을 방지하는지 검증하기 위한 테스트 항목입니다.

## 1. 정적 검증 항목

| ID | 테스트 항목 | 검증 방법 | 예상 결과 |
| :--- | :--- | :--- | :--- |
| K-V-1 | 로거 마스킹 로직 | Logger 구현부 확인 | 출력 문자열 내에서 알려진 비밀 값을 마스킹하는 필터 함수가 존재하고 적용되어 있다. |
| K-V-2 | 에러 객체 정제 | 에러 핸들러 확인 | 외부 API 에러 출력 시 헤더 또는 URL 내의 Key를 숨기는 처리 로직이 존재한다. |

## 2. 기능 테스트 항목

### K-T-1: 일반 로그 출력 마스킹

- **준비:** 가상의 API Key를 포함하는 문자열 구성 (예: `Using key: AIzaSyA...`)
- **실행:** Logger를 통해 해당 문자열 출력
- **예상 결과:** 콘솔에는 `Using key: ***` 형태로 마스킹되어 출력된다.

### K-T-2: 객체 덤프 보안

- **준비:** Credentials 객체를 `console.log` 또는 Logger에 전달
- **실행:** 로거가 객체를 처리하여 출력
- **예상 결과:** 객체 내의 `apiKey` 필드가 숨겨지거나 마스킹되어 출력된다.

### K-T-3: 강제 에러 유발 및 메시지 검사

- **준비:** 가상의 인증 정보가 포함된 설정으로 존재하지 않는 엔드포인트 호출 유도
- **실행:** 예외 발생시키고 에러 로그 확인
- **예상 결과:** 어떠한 에러 메시지나 stack trace 내에서도 실제 API Key 원문이 검출되지 않는다.

## 3. 검증 결과 요약

- **모든 항목 통과 시:** 실행 과정 중 API Key 유출을 완벽하게 차단함.
- **실패 항목 존재 시:** Logger 및 Error Handler의 텍스트 필터링 정규식 등을 점검한다.
